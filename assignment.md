## Цель проекта
Проект направлен на разработку телеграм-бота, который позволяет пользователю получать прогноз цен акций и рекомендации по торговым стратегиям. Пользователь вводит название компании и сумму для условной инвестиции, бот автоматически загружает исторические данные о стоимости акций, обучает несколько моделей временных рядов, выбирает наилучшую по метрикам качества и строит прогноз на ближайшие 30 дней.

В финале пользователь получает:

* график прогноза;
* оценку изменения цены акций относительно текущего дня;
* сводку с рекомендациями по дням покупки и продажи;
* расчёт потенциальной прибыли.
  
Дополнительно бот ведёт журнал логов, где фиксируются все ключевые параметры каждого запроса.

## Требования к данным
* Источник: Yahoo Finance.
* Данные загружаются за последние два года.
* Поддерживаемый формат хранения — CSV (при необходимости сохранения).
* Бот должен корректно работать при вводе любых популярных тикеров.
* Бот должен обрабатывать некорректно введённые команды или тикеры.

## Этапы выполнения
**Этап 1. Взаимодействие с пользователем**  
- Пользователь указывает в Telegram тикер компании (например, AAPL, MSFT) и сумму денег для условной инвестиции.
- Бот с помощью библиотеки yfinance или аналогичного API загружает котировки акций за последние два года.

**Этап 2. Обучение и выбор модели**
- Для прогнозирования реализуйте не менее трёх моделей разных классов:
    * Классическую ML-модель (например, Random Forest, Ridge Regression с лаговыми признаками).
    * Статистическую модель (например, ARIMA, ETS или Prophet).
    * Нейросетевую модель (например, LSTM, GRU или RNN для временных рядов).
- Модели обучаются на исторических данных, сравниваются по метрикам (RMSE, MAPE и другим) на тестовом множестве.
- Лучшая модель выбирается автоматически и используется для прогноза.

**Этап 3. Прогнозирование и визуализация**
- С помощью выбранной модели бот строит прогноз цен акций на 30 дней вперёд.
- Пользователю отправляется график с историей и предсказанными значениями.
- Дополнительно бот сообщает, насколько акции вырастут или упадут относительно текущего дня.

**Этап 4. Инвестиционные рекомендации**
- Бот анализирует предсказанный ряд:
    * выделяет дни с локальными максимумами (рекомендация — продавать);
    * определяет локальные минимумы (рекомендация — покупать).
- Исходя из введённой суммы, бот рассчитывает, какой условный доход можно было бы получить при следовании этим рекомендациям.
- В конце отправляется сводка: стратегия торговли и ориентировочная прибыль.

**Этап 5. Логирование**
- Каждый запрос пользователя фиксируется в отдельной строке текстового файла (например, logs.txt).
- Для каждой сессии сохраняются:
    * ID пользователя;
    * дата и время запроса;
    * тикер компании;
    * сумма, указанная пользователем;
    * модель, выбранная как наилучшая;
    * значение метрики (например, RMSE или MAPE);
    * рассчитанная ориентировочная прибыль.

## Ограничения и риски
* Недопустимо ограничиваться одной моделью: необходимо реализовать минимум три подхода.
* Выбор модели должен быть обоснован результатами на тестовом множестве, а не субъективно.
* Все расчёты должны выполняться в реальном времени после запроса пользователя.
* Бот не является финансовым инструментом, результаты носят исключительно учебный характер.
